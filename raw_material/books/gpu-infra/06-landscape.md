# 第六部分：生态与趋势

## 趋势

**三层架构分析模型**（替代"项目列表式"）：
- **控制面**：资源治理——配额、队列、优先级（定义"谁、多少、什么约束"）
- **数据平面**：抽象请求转化为隔离、可共享的设备访问（设备发现、资源分片）
- **工作负载层**：模型如何消费 GPU（推理并发、训练拓扑、显存动态）

**四大决策维度**：
1. 资源单位轴：整卡 → MIG 分片 → vGPU 配额 → 更高抽象
2. 隔离轴：软隔离 → 强隔离 → 可验证隔离
3. 形状匹配轴：碎片化来源识别（离散单位、请求形状、拓扑约束）
4. 治理轴：可解释性、可审计性、计费、基于 SLO 的交付

**四大趋势**：
1. **设备抽象进化**：从基础 Device Plugin 机制 → "资源语义层"，实现真正的多租户管理
2. **调度演进**：从"调度器中心" → "队列优先 + 运营资源池化"，策略分布在控制/数据/工作负载各层
3. **资源承诺化**：从简单分配 → 承诺 P99 延迟、完成时间、故障率
4. **异构硬件普及**：形状匹配调度器需求增加；异构加速多样性使整个生态演变为"能力市场 + 调度语言 + 运营系统"

**可持续评估模板**：对每个新项目——按层分类 → 识别变化的决策轴 → 建立验证指标（吞吐、P99、干扰系数、碎片化率）→ 在最小测试环境验证。
